# M7.1 - Reconciliation Loop: Implementation Plan

**Created**: 2026-01-07  
**Status**: Ready to Implement  
**Goal**: Background reconciliation loop to detect discrepancies between internal state and exchange state

---

## Overview

The reconciliation loop continuously compares our internal execution records against the exchange's actual order state to detect:
- Orders placed internally but missing on exchange (network failures)
- Orders on exchange but missing internally (manual intervention or crash recovery)
- Status mismatches (canceled vs open, filled vs pending)
- Orphaned orders that need investigation

**Key Principle**: Detection only, no auto-remediation (fail-safe approach for MVP)

### Phase 2: New Discrepancy Tracking Table

```sql
-- Detailed discrepancy log for investigation
create table if not exists reconciliation_discrepancy (
    discrepancy_id uuid primary key default gen_random_uuid(),
    execution_id uuid references execution_intent(execution_id), -- nullable for orphaned orders
    detected_at_utc timestamptz not null default now(),
    discrepancy_type text not null, -- MISSING_ON_EXCHANGE, ORPHANED_ON_EXCHANGE, STATUS_MISMATCH, etc.
    internal_state jsonb not null,  -- Our records
    exchange_state jsonb not null,  -- Exchange response
    details text,
    resolved boolean not null default false,
    resolved_at_utc timestamptz,
    resolution_notes text
);

create index idx_reconciliation_discrepancy_execution on reconciliation_discrepancy(execution_id);
create index idx_reconciliation_discrepancy_detected on reconciliation_discrepancy(detected_at_utc);
create index idx_reconciliation_discrepancy_unresolved on reconciliation_discrepancy(resolved) where resolved = false;
create index idx_reconciliation_discrepancy_type on reconciliation_discrepancy(discrepancy_type);
```

#### 5. **`ReconciliationService`** implementation
**File**: `src/Mvp.Trading.Execution/ReconciliationService.cs`

**Dependencies**:
- `IExecutionIntentStore` (query active executions)
- `IOrderReceiptStore` (query order receipts)
- `ITradingProvider` (query exchange open orders)
- `IReconciliationStore` (persist discrepancies)
- `ILogger<ReconciliationService>`

**Key Logic**:
1. Query all active execution intents (status != 'COMPLETED', 'FAILED')
2. For each execution, query order receipts
3. Call `ITradingProvider.GetOpenOrdersAsync()` to get exchange state
4. Compare internal orders vs exchange orders
5. Detect missing, orphaned, or mismatched orders
6. Persist discrepancies to database
7. Return summary result

public async Task<IReadOnlyList<ExecutionIntent>> GetActiveAsync(CancellationToken ct = default)
{
    const string sql = @"
        select execution_id, plan_id, mode, status, created_at_utc
        from execution_intent
        where status not in ('COMPLETED', 'FAILED')
        order by created_at_utc desc";
    
    await using var conn = await _dataSource.OpenConnectionAsync(ct);
    await using var cmd = new NpgsqlCommand(sql, conn);
    await using var reader = await cmd.ExecuteReaderAsync(ct);
    
    var results = new List<ExecutionIntent>();
    while (await reader.ReadAsync(ct))
    {
        results.Add(new ExecutionIntent(
            reader.GetGuid(0),
            reader.GetGuid(1),
            reader.GetString(2),
            reader.GetString(3),
            reader.GetFieldValue<DateTimeOffset>(4)
        ));
    }
    
    return results;
}
```

public async Task<IReadOnlyList<OrderReceipt>> GetByExecutionIdAsync(Guid executionId, CancellationToken ct = default)
{
    const string sql = @"
        select client_order_id, exchange_order_id, status, qty, price
        from order_receipt
        where execution_id = @execution_id
        order by created_at_utc";
    
    await using var conn = await _dataSource.OpenConnectionAsync(ct);
    await using var cmd = new NpgsqlCommand(sql, conn);
    cmd.Parameters.AddWithValue("execution_id", executionId);
    await using var reader = await cmd.ExecuteReaderAsync(ct);
    
    var results = new List<OrderReceipt>();
    while (await reader.ReadAsync(ct))
    {
        results.Add(new OrderReceipt(
            reader.GetString(0),
            reader.IsDBNull(1) ? null : reader.GetString(1),
            reader.GetString(2),
            reader.IsDBNull(3) ? null : reader.GetDecimal(3),
            reader.IsDBNull(4) ? null : reader.GetDecimal(4)
        ));
    }
    
    return results;
}
```

## Risks and Mitigations

| Risk | Impact | Mitigation |
|------|--------|------------|
| Exchange API rate limits exhausted | High | 60s polling interval, existing rate limiter |
| False positive discrepancies | Medium | Manual review required, no auto-remediation |
| Reconciliation loop crashes | High | Try-catch in worker, log errors, continue loop |
| Database connection failures | Medium | Retry logic, fail gracefully, alert on repeated failures |
| Large number of discrepancies | Low | `MaxDiscrepanciesToLog` config to prevent log spam |
