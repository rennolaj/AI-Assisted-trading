# M6.6 - Order Placement Tests

**Status**: ✅ Complete  
**Created**: 2026-01-07  
**Purpose**: Add integration tests for Kraken Futures order placement

## Motivation

During M6 completion review, we identified that while the order placement implementation exists (`KrakenFuturesTradingProvider.SendOrderAsync()`), there were **no tests** validating that orders are created correctly on the exchange. This creates risk that:
- Orders might fail silently in production
- API parameter formatting issues could go undetected
- Exchange API changes could break execution without warning

This was addressed by creating `tests/Mvp.Trading.Execution.Tests/` project.

## What Was Created

### New Test Project: `Mvp.Trading.Execution.Tests`

**Files:**
- `Mvp.Trading.Execution.Tests.csproj` - Test project configuration
- `KrakenFuturesTradingIntegrationTests.cs` - Integration tests for trading API
- `README.md` - Test documentation and usage guide

### Test Coverage

✅ **6 Integration Tests:**
1. `CheckApiKey_WithValidCredentials_ReturnsSuccess` - Validates API key and permissions
2. `SendOrder_LimitOrder_ReturnsOrderAck` - Places and cancels a limit order
3. `GetOpenOrders_ReturnsOrderList` - Retrieves open orders
4. `CancelOrder_WithValidOrderId_ReturnsSuccess` - Cancels a specific order
5. `CancelAllOrdersAfterAsync_WithValidTimeout_ReturnsSuccess` - Configures dead-man's switch (60s)
6. `CancelAllOrdersAfterAsync_DisableWithZero_ReturnsSuccess` - Disables dead-man's switch

### Test Behavior

- **Disabled by default** - Tests skip unless `KRAKEN_FUTURES_TRADING_TESTS=1` is set
- **Requires credentials** - `KRAKEN_FUTURES_API_KEY` and `KRAKEN_FUTURES_API_SECRET` environment variables
- **Demo API only** - Defaults to `https://demo-futures.kraken.com/derivatives`
- **Safe execution** - Orders placed far from market price to avoid fills, all orders cleaned up in test teardown
- **CI-friendly** - Tests pass (skip) in CI without credentials

## Testing Strategy

### Local Testing (Optional)

Developers can run integration tests locally against Kraken Demo API:

```bash
export KRAKEN_FUTURES_TRADING_TESTS=1
export KRAKEN_FUTURES_API_KEY="your-demo-api-key"
export KRAKEN_FUTURES_API_SECRET="your-demo-api-secret"
dotnet test tests/Mvp.Trading.Execution.Tests/
```

### CI Testing

Tests automatically skip in CI (no credentials configured). This is intentional:
- Keeps CI fast
- No external dependencies required
- Integration tests can be run on-demand or scheduled

### Production Validation

Before deploying to production:
1. Run integration tests against Kraken Demo environment
2. Verify all 6 tests pass
3. Smoke test with real (demo) orders
