## Key Metrics to Track

### Alert Processing (7 metrics)
1. `alerts_received_total` - Counter with labels: exchange, symbol
2. `alerts_processed_total` - Counter with labels: outcome (accepted, rejected_indicator, rejected_elliott, rejected_llm, error)
3. `alert_processing_duration_seconds` - Histogram
4. `queue_depth` - Gauge (current alerts in Redis queue)

### Execution (8 metrics)
5. `executions_processed_total` - Counter with labels: outcome (filled, placed, rejected_risk, rejected_heartbeat, error)
6. `execution_duration_seconds` - Histogram with labels: stage (total, llm_adjudication, order_placement)
7. `orders_placed_total` - Counter with labels: direction (LONG/SHORT), order_type (LIMIT/MARKET)
8. `orders_cancelled_total` - Counter with labels: reason
9. `orders_filled_total` - Counter with labels: symbol, direction
10. `active_trades` - Gauge (current open positions)

### Risk Events (2 metrics)
11. `stop_loss_triggered_total` - Counter with labels: symbol
12. `take_profit_hit_total` - Counter with labels: symbol, target

### Errors (2 metrics)
13. `errors_total` - Counter with labels: component, error_type
14. `api_errors_total` - Counter with labels: provider, endpoint, error_code

### Reconciliation (1 metric)
15. `reconciliation_discrepancies` - Gauge (current unresolved discrepancies)

## Technology Stack
- **Metrics SDK**: OpenTelemetry 1.9.0
- **Exporter (API)**: OpenTelemetry.Exporter.Prometheus.AspNetCore
- **Exporter (Worker)**: OpenTelemetry.Exporter.Prometheus.HttpListener
- **Instrumentation**: AspNetCore, HttpClient
- **Health Checks**: AspNetCore.HealthChecks.Redis, AspNetCore.HealthChecks.Npgsql
- **Metrics Backend**: Prometheus (latest)
- **Visualization**: Grafana (latest)

## Health Check Endpoints
- `GET /health` - Overall health (all checks, JSON response)
- `GET /health/ready` - Readiness probe (Kubernetes-compatible)
- `GET /health/live` - Liveness probe (Kubernetes-compatible)

## Prometheus Endpoints
- API: `http://localhost:8080/metrics` (AspNetCore exporter)
- Worker: `http://localhost:9464/metrics` (HttpListener exporter)

## Risks & Mitigations
- **Risk**: Performance impact from metrics collection
  - **Mitigation**: OpenTelemetry is designed for low overhead, use sampling if needed
- **Risk**: Port conflicts (9090, 3000, 9464)
  - **Mitigation**: Document ports, make configurable via docker-compose
- **Risk**: Metrics cardinality explosion (too many unique label combinations)
  - **Mitigation**: Carefully limit label values, avoid unbounded strings
