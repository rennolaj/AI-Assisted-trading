# M6 Note: Partial Targets Requirement

## Summary
- Requirement: each trade should include **at least 3 partial targets**.
- This is now enforced in `TradePlan` and generated by the builder.

## Current Implementation
- `TradePlan` includes `TakeProfitTargets` (price, quantity, reason).
- Schema requires `takeProfitTargets` with `minItems: 3`.
- Builder allocates targets at 1R/2R/3R with 50/30/20 split.
- Target quantities are step-aligned and must satisfy min-qty constraints.

## Open Decisions
- Whether target allocation and R-multiples should be config-driven.
- Whether strategies may override the default 3-target requirement.

## Next Step (Optional)
Move target allocation (multiples + split) into config for flexibility.
